## DoD IL4 Landing Zone Hub-Spoke Architecture on Azure Government

If you're architecting cloud infrastructure for a DoD Impact Level 4 workload, you already know the stakes: ITAR data, CUI, strict network segmentation, and an ATO timeline that punishes architectural mistakes made early. The hub-spoke topology isn't just a best practice in this context — it's a compliance requirement wrapped in an architectural pattern. Let me walk you through exactly how I design, deploy, and harden a DoD IL4 landing zone using Azure Government (GCC High), following NIST SP 800-53 Rev 5 controls and the DoD Cloud Computing Security Requirements Guide (SRG).

---

## What Hub-Spoke Means in a DoD IL4 Context

A hub-spoke architecture in Azure Government separates shared security services (the hub) from mission workloads (the spokes). The hub VNet hosts your Azure Firewall Premium, Azure Bastion, centralized DNS, and your connection back to on-prem via ExpressRoute or VPN. Spoke VNets are peered to the hub — never to each other — creating a forced-traffic-inspection model where all east-west and north-south traffic transits the firewall.

For IL4, this matters because:

- **SC-7 (Boundary Protection)** requires managed interfaces and traffic filtering at network boundaries
- **AC-4 (Information Flow Enforcement)** demands that CUI never traverses uncontrolled paths
- **SI-4 (System Monitoring)** requires centralized logging of all network flows
- **CM-6 (Configuration Settings)** enforces hardened baselines across all compute resources

The hub also becomes your centralized **Azure Policy** and **Microsoft Defender for Cloud** management point, which directly supports your continuous monitoring obligations under FedRAMP High and the DoD SRG IL4 baseline.

I deploy Azure Firewall **Premium** SKU — not Standard — because IL4 requires TLS inspection and IDPS (Intrusion Detection and Prevention System), which are only available in Premium. This directly maps to **SI-3 (Malicious Code Protection)** and **SC-5 (Denial of Service Protection)**.

## Step 3: Spoke VNet Peering with UDR Enforcement

Every spoke must peer to the hub with **UseRemoteGateways** enabled and a User Defined Route (UDR) forcing all traffic through the Azure Firewall private IP. This is non-negotiable for IL4 — uncontrolled routing paths will sink your ATO.

<CodeBlock className="custom-code-block" liveEditor codeInstances={[{ code: `# Deploy spoke VNet and peer to hub

HUB_VNET_ID="/subscriptions/<hub-sub-id>/resourceGroups/rg-hub-il4/providers/Microsoft.Network/virtualNetworks/vnet-hub-il4-usgva-001"
FIREWALL_PRIVATE_IP="10.0.1.4"
SPOKE_RG="rg-spoke-workload-il4"
LOCATION="usgovvirginia"

# Create Spoke VNet
az network vnet create \\
  --name vnet-spoke-workload-il4 \\

## Step 4: Azure Policy Assignments for IL4 Compliance

Policy-as-code is your enforcement layer. I assign the **DoD IL4** built-in initiative (or the closest proxy, NIST SP 800-53 Rev 5 initiative) at the management group level. This ensures every subscription inherits the baseline.

Key policies I enforce at the MG level:

| Policy | NIST Control | Effect |
|---|---|---|
| Require Azure Firewall on VNets | SC-7 | Audit/Deny |
| Deny public IP on NICs | SC-7(3) | Deny |
| Require CMK on storage accounts | SC-28 | Deny |
| Require HTTPS-only on Storage | SC-8 | Deny |
| Enforce diagnostic settings to Log Analytics | AU-2, AU-9 | DeployIfNotExists |
| Deny RDP/SSH from internet | AC-17 | Deny |

```powershell
# PowerShell - Assign NIST 800-53 Rev 5 initiative at MG scope
Connect-AzAccount -Environment AzureUSGovernment

$mgId = "dodil4-root"
$initiativeId = "/providers/Microsoft.Authorization/policySetDefinitions/179d1daa-458f-4e47-8086-2a68d0d6c38f"

# Get or create Log Analytics workspace resource ID for diagnostic settings
$lawId = "/subscriptions/<sub-id>/resourceGroups/rg-hub-il4/providers/Microsoft.OperationalInsights/workspaces/law-il4-usgva-001"

New-AzPolicyAssignment `
  -Name "nist-80053-r5-il4-root" `
  -DisplayName "NIST SP 800-53 Rev 5 - DoD IL4 Root" `
  -PolicySetDefinition (Get-AzPolicySetDefinition -Id $initiativeId) `
  -Scope "/providers/Microsoft.Management/managementGroups/$mgId" `
  -PolicyParameterObject @{
    "logAnalyticsWorkspaceIdForVMAgents" = @{ "value" = $lawId }
  } `
  -IdentityType "SystemAssigned" `
  -Location "usgovvirginia"
```
